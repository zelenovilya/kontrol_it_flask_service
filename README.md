# Контроль ИТ: веб-сервис учета и обработки заявок

Веб-сервис разработан для ООО «Контроль ИТ» как учебный сетевой ресурс на Python Flask. Проект автоматизирует регистрацию заявок клиентов на ИТ-обслуживание, назначение специалистов, контроль статусов, работу с файлами, формирование актов `.docx` и отчетов `.xlsx`.

Проект не использует CMS, готовые CMS-плагины или конструкторы сайтов. Интерфейс, модели данных, маршруты и шаблоны реализованы в коде проекта.

Автор проекта: Зеленов Илья Денисович.

## Стек

- Python 3.9+

- HTML, CSS, JavaScript

## Локальный запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python main.py
```

Для Windows активация окружения выполняется командой:

```bash
.venv\Scripts\activate
```

После запуска приложение доступно по адресу:

```text
http://127.0.0.1:5000
```

## Роли пользователей

- Клиент создает заявки, прикрепляет файлы, добавляет комментарии, просматривает свои обращения и скачивает документы.
- Специалист работает с назначенными заявками, меняет статус, добавляет комментарии, прикрепляет файлы, формирует акт `.docx` и скачивает отчет `.xlsx`.
- Администратор управляет пользователями, ролями, клиентами, сотрудниками, услугами, категориями, статусами, приоритетами, заявками, материалами базы знаний, обратной связью, документами и отчетами.

## База данных

Учебная SQLite-база с тестовыми данными лежит в репозитории:

```text
data/kontrol_it.db
```

В базе 14 таблиц: `roles`, `users`, `clients`, `employees`, `service_categories`, `services`, `ticket_statuses`, `ticket_priorities`, `tickets`, `ticket_comments`, `attachments`, `documents`, `feedback_messages`, `articles`.

Тестовые данные включают пользователей трех ролей, клиентов, сотрудников, категории услуг, услуги, статусы, приоритеты, заявки, комментарии, обращения обратной связи, статьи, новости, вложения и документ.

Если файл базы удалить, приложение при запуске автоматически создаст новую базу и заполнит ее начальными данными из `app/seed.py`.

## Работа с файлами

Клиент может прикрепить файл при создании заявки. Специалист может прикрепить файл при обработке заявки. В карточке заявки отображаются имя файла, дата загрузки и пользователь, который добавил файл.

Скачивание вложений реализовано в `app/routes/downloads.py`:

- клиент скачивает файлы только по своим заявкам;
- специалист скачивает файлы только по назначенным ему заявкам;
- администратор скачивает файлы по всем заявкам;
- если файл отсутствует на диске, пользователь получает предупреждение и возвращается в карточку заявки.

Тестовые вложения лежат в папке `uploads/`.

## Документы и отчеты

Формирование акта `.docx` реализовано в маршруте специалиста `generate_act` в `app/routes/specialist.py`. Сформированный документ сохраняется в базе как запись `Document` и доступен для скачивания через защищенный маршрут.

Формирование отчетов `.xlsx` реализовано через openpyxl:

- администратор скачивает общий отчет по заявкам;
- специалист скачивает отчет по своим заявкам.

Тестовый DOCX-документ лежит в `generated/documents/`.
