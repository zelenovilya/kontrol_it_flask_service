# Тестовый чек-лист

## Публичная часть

- [ ] Главная страница открывается по адресу `/`.
- [ ] Публичные страницы «О компании», «Услуги», «Тарифы», «Как подать заявку», «Преимущества», FAQ, «База знаний», «Новости», «Контакты», «Справка» открываются без авторизации.
- [ ] Страница статьи базы знаний открывается из списка материалов.
- [ ] Страница новости открывается из списка новостей.
- [ ] Форма обратной связи сохраняет сообщение и показывает уведомление об отправке.
- [ ] Регистрация клиента создает пользователя с ролью «Клиент».

## Авторизация и доступ

- [ ] Вход клиента работает с учетной записью `client / client123`.
- [ ] Вход специалиста работает с учетной записью `specialist / spec123`.
- [ ] Вход администратора работает с учетной записью `admin / admin123`.
- [ ] Неавторизованный пользователь не может открыть закрытые страницы клиента, специалиста и администратора.
- [ ] Клиент не может открыть административную панель.
- [ ] Специалист не может открыть административную панель.

## Кабинет клиента

- [ ] Клиент видит главную страницу личного кабинета.
- [ ] Клиент открывает список «Мои заявки».
- [ ] Клиент создает заявку с услугой, приоритетом и описанием.
- [ ] Клиент прикрепляет файл при создании заявки.
- [ ] Клиент открывает карточку своей заявки.
- [ ] Клиент добавляет комментарий к своей заявке.
- [ ] Клиент видит имя файла, дату загрузки и автора вложения.
- [ ] Клиент скачивает файл по своей заявке.
- [ ] Клиент видит документы по своим заявкам.
- [ ] Клиент открывает профиль и справку клиента.

## Кабинет специалиста

- [ ] Специалист видит главную страницу кабинета.
- [ ] Специалист открывает список новых заявок.
- [ ] Специалист открывает назначенную заявку.
- [ ] Специалист меняет статус заявки.
- [ ] Специалист добавляет комментарий.
- [ ] Специалист прикрепляет файл к заявке.
- [ ] Специалист скачивает файл по назначенной заявке.
- [ ] Специалист открывает историю работ по заявке.
- [ ] Специалист формирует акт `.docx`.
- [ ] Специалист видит документ в разделе документов.
- [ ] Специалист формирует отчет `.xlsx`.
- [ ] Специалист открывает справку специалиста.

## Административная панель

- [ ] Администратор видит dashboard со статистикой.
- [ ] Администратор открывает раздел пользователей.
- [ ] Администратор добавляет пользователя.
- [ ] Администратор редактирует пользователя.
- [ ] Администратор блокирует и разблокирует учетную запись.
- [ ] Администратор открывает роли, клиентов и сотрудников.
- [ ] Администратор открывает услуги и категории услуг.
- [ ] Администратор открывает статусы и приоритеты заявок.
- [ ] Администратор видит все заявки.
- [ ] Администратор открывает карточку заявки.
- [ ] Администратор назначает специалиста на заявку.
- [ ] Администратор меняет статус заявки.
- [ ] Администратор добавляет комментарий к заявке.
- [ ] Администратор видит файлы и документы в карточке заявки.
- [ ] Администратор скачивает файл из карточки заявки.
- [ ] Администратор видит сообщения обратной связи.
- [ ] Администратор отмечает сообщение обратной связи обработанным.
- [ ] Администратор добавляет материал базы знаний или новость.
- [ ] Администратор видит список документов.
- [ ] Администратор формирует отчет `.xlsx`.
- [ ] Администратор открывает справку администратора.

## Данные и устойчивость

- [ ] После перезапуска приложения ранее созданные данные сохраняются в SQLite.
- [ ] В репозитории присутствует база `data/kontrol_it.db` с тестовыми данными.
- [ ] В базе не менее 10 таблиц.
- [ ] При отсутствии файла вложения система показывает понятное предупреждение.
- [ ] Хлебные крошки отображаются на публичных и закрытых страницах.
- [ ] В подвале каждой страницы указано ФИО автора: Зеленов Илья Денисович.
